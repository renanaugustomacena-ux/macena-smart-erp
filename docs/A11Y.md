# Accessibility — SmartERP

**Target:** WCAG 2.1 AA for the landing page (Italian), the authenticated dashboard (Italian/English toggle), and the printed PDF invoice output.
**Methodology:** Manual keyboard + screen-reader walk-through each release; automated axe-core in CI.

---

## 1. Scope

| Surface | Profile |
|---------|---------|
| Landing page | WCAG 2.1 AA (public site, key for marketing & EN/IT switching) |
| Dashboard (`/`) | WCAG 2.1 AA |
| Invoice PDF | PDF/UA-1 (ISO 14289-1) when PDFs are generated (roadmap) |
| Swagger UI (dev only) | Best-effort; not customer-visible |

---

## 2. Landing Page (`landing-page/index.html`)

| Criterion | Implementation |
|-----------|----------------|
| 1.1.1 Non-text content | `alt` attributes on images; `aria-hidden="true"` for decorative SVG marks. |
| 1.3.1 Info and relationships | Semantic `<nav>`, `<main>`, `<section aria-labelledby>`, `<footer>`. |
| 1.4.3 Contrast | Body text ≥ 4.5:1 contrast ratio; Verona-primary `#1e3a8a` on white = 9.2:1. |
| 2.1.1 Keyboard | "Skip to main content" link (`.skip-link`). Tab order follows DOM order. |
| 2.4.4 Link purpose | Every link has a descriptive label (no "click here"). |
| 3.1.1 Language of page | `<html lang="it">`. |
| 3.3.2 Labels or instructions | Form fields would use `<label for="...">` when a sign-up form lands. |
| 4.1.2 Name, role, value | ARIA labels on icon buttons. |

Automated check: `npx @axe-core/cli http://localhost:3000/` — expected 0 serious, 0 critical.

---

## 3. Dashboard (Next.js)

- `next/font` for variable-font loading avoids FOIT/FOUT.
- Focus visible via Tailwind `focus-visible:ring-2 ring-verona-600`.
- Modal dialogs (when added) use `<dialog>` element with `::backdrop` for native keyboard-trap behaviour.
- Notification toasts use `role="status"` (polite) for non-critical and `role="alert"` (assertive) for errors.
- Data tables use `<th scope="col">` and `aria-sort`.
- Charts (Recharts) provide a `<caption>` and tabular fallback for screen readers.

## 4. Colour Palette

| Token | Hex | Role | Min contrast |
|-------|-----|------|--------------|
| `primary` | `#1e3a8a` | Primary links / CTAs | 9.2:1 on white |
| `accent` | `#be185d` | Highlighted errors | 6.1:1 on white |
| `success` | `#047857` | Success badges | 5.8:1 on white |
| `warn` | `#b45309` | Warnings | 5.3:1 on white |
| `body` | `#111827` | Body text | 16.1:1 on white |
| `subtle` | `#4b5563` | Muted text | 8.4:1 on white |

All text tokens meet WCAG AA (4.5:1) on the white background.

## 6. Language Policy

- Landing page and marketing surfaces: **100% Italian** (G16 gate).
- Invoices (PDF + XML): Italian (FatturaPA requires it).
- In-app UI: Italian default; English opt-in for Enterprise tenants with multilingual teams.
- Error messages: Italian default; backend API sends canonical English keys in `error.code`, human message in `error.message` localised to the authenticated user's preferred language.
